## Context

output for a ribbed hollow planter. Two independent fixes are needed, ordered by
priority.

## Fix 1 — Parser regex (PRIORITY: HIGH, ~1–2 hrs work)

**File:** `app/parsers/creality.py`

### What's broken

The existing patterns target an older Creality format. V7 (OrcaSlicer-derived) changed:

| Field | Old format (parser expects) | New V7 format (actual) |
|---|---|---|
| Filament | `; filament used: 486.9g` | `; filament used [g] = 0.00, 0.00, 0.00, 349.12` |
| Time | `; Estimated printing time...: 9h5m` | `; estimated printing time (normal mode) = 13h 28m 58s` |

The parser raises `ValueError` on any V7 G-code file — it never returns a result.

### Required changes to `app/parsers/creality.py`

**1. Detection markers** — add `orca` so OrcaSlicer files are also handled:
```python
_DETECTION_MARKERS: tuple[str, ...] = (
    "creality",
    "crealityprint",
    "orca",
)
```

**2. Filament pattern** — handle both old (single value) and new (multi-extruder array)
formats. Sum all non-zero values in the array:
```python
# Old: ; filament used: 486.9g
_FILAMENT_LEGACY_PATTERN = re.compile(
    r";\s*filament used\s*:\s*([\d.]+)\s*g", re.IGNORECASE
)

# New V7/Orca: ; filament used [g] = 0.00, 0.00, 0.00, 349.12
_FILAMENT_GRAMS_PATTERN = re.compile(
    r";\s*filament used \[g\]\s*=\s*([\d.,\s]+)", re.IGNORECASE
)
```

Extraction logic:
```python
filament_grams: float | None = None

m = _FILAMENT_GRAMS_PATTERN.search(text)
if m:
    values = [float(v.strip()) for v in m.group(1).split(",")]
    filament_grams = sum(v for v in values if v > 0)
else:
    m = _FILAMENT_LEGACY_PATTERN.search(text)
    if m:
        filament_grams = float(m.group(1))

if filament_grams is None:
    raise ValueError("CrealityGcodeParser: filament usage not found")
```

**3. Time pattern** — handle both old (`9h5m`) and new (`13h 28m 58s`) formats,
using `=` or `:` as separator:
```python
# New V7/Orca: ; estimated printing time (normal mode) = 13h 28m 58s
_TIME_PATTERN = re.compile(
    r";\s*estimated printing time[^=:]*[=:]\s*"
    r"(?:(\d+)h\s*)?(?:(\d+)m\s*)?(?:(\d+)s)?",
    re.IGNORECASE,
)
```

Update `_parse_time_to_seconds` to accept seconds:
```python
def _parse_time_to_seconds(hours: str | None, minutes: str | None, seconds: str | None) -> int:
    h = int(hours) if hours else 0
    m = int(minutes) if minutes else 0
    s = int(seconds) if seconds else 0
    return h * 3600 + m * 60 + s
```

Call site:
```python
print_time_seconds = _parse_time_to_seconds(
    time_match.group(1), time_match.group(2), time_match.group(3)
)
```

**4. New fields to extract and add to `PrintEstimate`**

Add to `app/parsers/base.py` — `PrintEstimate` dataclass:
```python
@dataclass
class PrintEstimate:

    print_time_seconds: int
    slicer_name: str
    source: str = "gcode"
    layer_height: Optional[float] = None
    nozzle_diameter: Optional[float] = None
    # New fields
    wall_count: Optional[int] = None
    infill_percentage: Optional[float] = None
    speed_wall_outer_mm_s: Optional[float] = None
    speed_wall_inner_mm_s: Optional[float] = None
    speed_infill_mm_s: Optional[float] = None
    filament_volume_cm3: Optional[float] = None
```

Add to `creality.py`:
```python
_WALL_LOOPS_PATTERN   = re.compile(r";\s*wall_loops\s*=\s*(\d+)", re.IGNORECASE)
_INFILL_PATTERN       = re.compile(r";\s*sparse_infill_density\s*=\s*([\d.]+)", re.IGNORECASE)
```

## Fix 2 — Geometry normalizer (PRIORITY: MEDIUM, ~3–4 hrs work)

**Files:** `app/services/print_normalizer.py`, `app/routers/products.py` (or wherever
`estimate-from-geometry` is wired), `app/parsers/base.py` (already updated above)

### Root causes

**Top/bottom overestimate (+62 g):**  
`footprint_area = x * y` uses full bounding rectangle. Hollow/circular shapes have
much less actual bottom material. Fix: derive cross-sectional area from volume and
height instead of bounding box.

```python
# Current (wrong for hollow/circular shapes):
footprint_area = x * y

# Better approximation:
# The average cross-sectional area of a solid = volume / height
# For a hollow shell this still over-counts, but is far better than x*y
footprint_area = volume_mm3 / z
```

For the planter: `639558 / 187 = 3419 mm²` vs bounding box `35920 mm²` — much
closer to reality.

**Wall underestimate (−55 g):**  
`perimeter_length = 2 * (x + y)` is one bounding rectangle per layer. Ribbed/curved
shapes have far more perimeter. Fix: use lateral surface area if FreeCAD provides it.

FreeCAD exposes `shape.Area` (total surface area). Pass it in the API payload:

In `InitGui.py` (FreeCAD workbench), add to `call_api()` payload:
```python
payload["surface_area_mm2"] = float(obj.Shape.Area)
```

In `print_normalizer.py`:
```python
def normalize_from_geometry(
    volume_mm3: float,
    dimensions_mm: Optional[dict[str, float]],
    density_g_cm3: float,
    profile: ProfileParams,
    color_changes: int = 0,
    surface_area_mm2: Optional[float] = None,   # NEW
) -> NormalizationResult:
```

Perimeter calculation when surface area is available:
```python
if surface_area_mm2 is not None:
    # lateral_area = total_area minus estimated top + bottom disks
    # footprint from volume/height is better than x*y for hollow shapes
    footprint_area = volume_mm3 / z
    lateral_area = max(0.0, surface_area_mm2 - 2.0 * footprint_area)
    extrusion_width = profile.nozzle_diameter_mm * profile.extrusion_width_factor
    perimeter_volume = lateral_area * profile.wall_count * extrusion_width
else:
    # fallback: bounding box rectangle
    perimeter_length = 2.0 * (x + y)
    perimeter_volume = (
        perimeter_length * profile.wall_count
        * extrusion_width * profile.layer_height_mm * layer_count
    )
```

Top/bottom with volume-derived footprint:
```python
footprint_area = volume_mm3 / z   # use everywhere, not x*y
top_bottom_volume = (
    footprint_area
    * (profile.top_layers + profile.bottom_layers)
    * profile.layer_height_mm
)
```

**Print time underestimate (−87%):**  
Single `volumetric_flow_rate_mm3s` ignores wall vs infill speed difference and
travel time. Fix: add per-speed-type fields to `ProfileParams` and compute two
time terms.

Add to `ProfileParams` dataclass:
```python
@dataclass
class ProfileParams:
    nozzle_diameter_mm: float
    layer_height_mm: float
    wall_count: int
    infill_percentage: float
    top_layers: int
    bottom_layers: int
    extrusion_width_factor: float
    volumetric_flow_rate_mm3s: float        # kept as fallback
    purge_mass_per_change_g: float
    speed_wall_mm_s: Optional[float] = None    # NEW — outer wall speed
    speed_infill_mm_s: Optional[float] = None  # NEW — sparse infill speed
    travel_overhead_factor: float = 1.15        # NEW — accounts for travel/accel (~12-15%)
```

Print time calculation:
```python
if profile.speed_wall_mm_s and profile.speed_infill_mm_s:
    # Per-type time using linear speeds and extrusion cross-section area
    extrusion_area = extrusion_width * profile.layer_height_mm  # mm²
    wall_seconds = perimeter_volume / (profile.speed_wall_mm_s * extrusion_area)
    infill_seconds = (infill_volume + top_bottom_volume) / (profile.speed_infill_mm_s * extrusion_area)
    print_hours = ((wall_seconds + infill_seconds) * profile.travel_overhead_factor) / 3600.0
else:
    # Legacy fallback
    total_extruded = perimeter_volume + infill_volume + top_bottom_volume
    print_hours = (total_extruded / profile.volumetric_flow_rate_mm3s) / 3600.0
```

### Print profile schema changes

Add `speed_wall_mm_s` and `speed_infill_mm_s` to the `print_profiles` table.

In `app/models.py` — `PrintProfile` model, add columns:
```python
speed_wall_mm_s    = Column(Float, nullable=True)
speed_infill_mm_s  = Column(Float, nullable=True)
```

In `app/schemas.py` — `PrintProfileCreate` / `PrintProfileRead`, add fields:
```python
speed_wall_mm_s:   Optional[float] = None
speed_infill_mm_s: Optional[float] = None
```

**Default values to seed for the "0.4mm Standard, 15% infill" profile:**

- `speed_infill_mm_s = 250`
